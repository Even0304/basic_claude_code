## 快速开始

### 1️⃣ 验证安装

```bash
python test_smoke.py
```

应该看到所有测试通过 ✅

### 2️⃣ 看并行执行的演示

```bash
kimi-code demo parallel
```

这会在你的项目目录运行 4 个分析任务，**同时执行**：
- 统计 Python 文件个数
- 统计代码总行数
- 列出项目依赖
- 查找所有 TODO 注释

**时间**: 只需要最慢的单个任务的时间（4倍加速！）

### 3️⃣ 启动交互模式

```bash
kimi-code
```

然后你就可以开始问问题了：

```
> 这个项目有多少个 Python 文件？

> 分析代码结构

> /tools          # 查看可用工具

> /help           # 查看所有命令

> /exit           # 退出
```

kimi-code demo parallel

# 可以指定目录
kimi-code demo parallel --target /path/to/project

## 并行执行

### 什么是并行执行？

kimi-code 最强大的功能是 **task 工具**，它可以同时运行多个任务：

```
用户: "统计文件、计算行数、找 TODO、列出依赖 - 同时做"
    ↓
Agent 识别出这 4 个任务是独立的
    ↓
使用 task 工具创建 4 个子 Agent
    ↓
4 个 Agent 同时运行（asyncio.gather）
    ↓
结果汇总呈现

时间: 只需最慢任务的时间（~10秒）
而不是顺序执行（~40秒）
```

### 并行的好处

| 任务 | 顺序执行 | 并行执行 | 加速 |
|------|---------|---------|------|
| 4 个文件扫描 | 40 秒 | 10 秒 | 4x |
| 3 个数据分析 | 30 秒 | 10 秒 | 3x |
| 5 个代码检查 | 50 秒 | 10 秒 | 5x |

### 如何触发并行执行

在提问中明确提到**同时/并行/并发**：

```
> 并行做这些事：
  1. 统计 Python 文件
  2. 计算总行数
  3. 找出所有 import
  4. 列出依赖

> 同时分析三个模块

> 并发执行以下操作...
```

## 工具列表

kimi-code 提供 7 个工具：

### 1️⃣ bash - 执行命令

```
用途：运行任何 shell 命令
示例：
  > 列出当前目录的所有文件
  > 查看 git 状态
  > 运行测试
```

### 2️⃣ read - 读取文件

```
用途：读取文件内容
示例：
  > 读取 requirements.txt 中的内容
  > 显示 main.py 的代码
  > 查看配置文件
```

### 3️⃣ write - 创建/写入文件

```
用途：创建新文件或覆盖文件
示例：
  > 创建一个 hello.py 文件
  > 保存代码到 test.py
```

### 4️⃣ edit - 编辑文件

```
用途：精确替换文件中的内容
示例：
  > 把代码中的 TODO 改成 DONE
  > 更新变量名
  > 修改配置值
```

### 5️⃣ glob - 查找文件

```
用途：按模式查找文件
示例：
  > 找出所有 .py 文件
  > 列出所有测试文件
  > 找出 src 目录下的所有 Python 文件
```

### 6️⃣ grep - 搜索内容

```
用途：在文件中搜索文本或正则表达式
示例：
  > 找出所有 TODO 注释
  > 搜索特定函数的定义
  > 找出所有 import 语句
```

### 7️⃣ task - 并行执行（关键！）

```
用途：同时运行多个独立任务
示例：
  > 并行做这些：
    1. 统计文件数
    2. 计算代码行数
    3. 列出依赖
    4. 找出 TODO
```

## 实际使用例子

### 例子 1：分析项目结构

```bash
kimi-code run "分析这个项目，告诉我有多少个 Python 文件、代码总行数、主要依赖是什么"
```

Agent 会：
- 使用 glob 找出所有 .py 文件
- 使用 bash 统计代码行数
- 使用 read 读取 requirements.txt/pyproject.toml

### 例子 2：并行分析（推荐）

```bash
kimi-code run "并行分析项目：统计 Python 文件、计算总行数、列出依赖、找出所有 TODO 注释"
```

Agent 会：
- 使用 task 工具同时启动 4 个子 Agent
- 4 个子 Agent 并行工作
- 汇总结果返回

**时间：约 10 秒（而不是 40 秒）**

### 例子 3：代码修改

```bash
kimi-code
> 把代码中所有的 print() 换成 logger.info()
> /exit
```

Agent 会：
- 使用 grep 找出所有 print 语句
- 使用 edit 逐个替换

### 例子 4：创建新代码

```bash
kimi-code
> 创建一个 hello.py，内容是 print("Hello, World!")
> 检查一下文件是否创建成功
> /exit
```

## 常见问题

### Q1：如何快速开始？

```bash
# 1. 看演示（最简单）
kimi-code demo parallel

# 2. 或者交互模式
kimi-code

# 3. 或者单次命令
kimi-code run "统计 Python 文件"
```

### Q2：怎样触发并行执行？

在你的问题中使用这些词：
- "并行"
- "同时"
- "并发"
- "同步执行这些"

例如：
```
> 并行计算：文件个数、代码行数、依赖列表
> 同时做三件事...
```

### Q3：工具会自动选择吗？

是的！Agent 会根据你的问题自动选择合适的工具：
- 需要执行命令？ → bash
- 需要读文件？ → read
- 需要搜索？ → grep/glob
- 需要并行？ → task

### Q4：可以修改已有代码吗？

可以！使用 edit 工具做精确替换：

```
> 把第 5 行的 "old_text" 改成 "new_text"
```

### Q5：如何列出所有可用工具？

在交互模式下：
```
> /tools
```

### Q6：如何清除对话历史？

在交互模式下：
```
> /clear
```

### Q7：如何知道当前用的是哪个模型？

在交互模式下：
```
> /model
显示：moonshotai/kimi-k2.5
```

### Q8：能查看对话历史吗？

在交互模式下：
```
> /history
```

### Q9：出错了怎么办？

1. 添加 `--debug` 查看详细信息：
   ```bash
   kimi-code --debug
   ```

2. 检查 `.env` 文件配置是否正确

3. 运行 smoke tests 验证：
   ```bash
   python test_smoke.py
   ```

### Q10：可以用 Claude 而不是 Kimi 吗？

可以！编辑 `.env` 文件：
```
PROVIDER=claude
ANTHROPIC_API_KEY=your-key-here
```

然后运行：
```bash
kimi-code --provider claude
```

## 提示和技巧

### 💡 提示 1：充分利用并行执行

当有多个独立任务时，明确说"并行"或"同时"，加速 4 倍！

```
好的做法:
> 并行分析：统计文件、计算行数、列出依赖、找 TODO

不好的做法:
> 统计文件
> 计算行数
> 列出依赖
> 找 TODO
```

### 💡 提示 2：使用 /clear 重新开始

如果对话变得混乱，使用 `/clear` 重新开始：

```
> /clear
> 现在开始一个新的分析...
```

### 💡 提示 3：指定工具集

如果要减少工具数量（提高速度）：

```bash
kimi-code run "find all Python files" --tools minimal
```

### 💡 提示 4：查看完整帮助

```bash
kimi-code --help
```
